# AI Handoff Documentation

Background context for a model (or human) with zero prior context on this repo.

**Reading order:**

1. **ARCHITECTURE.md** — how the library and docs site actually work; invariants; dragons.
   Read before any code change.
2. **CONVENTIONS.md** — mechanical style rules. Read before writing code.
3. **COMPONENT-FACTORY.md** — step-by-step playbook to ship a new component.
4. **QUALITY-BAR.md** — pass/fail rubrics + the mechanical pre-release checklist.
5. **DECISION-LOG.md** — why things are the way they are; when to revisit each decision.

Ground rule these documents assume: when a doc and the code disagree, trust the code and fix
the doc in the same PR.
