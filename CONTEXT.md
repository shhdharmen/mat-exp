# mat-expressive Docs Site

The documentation website for the `@ngm-dev/mat-expressive` Angular component library. It is a separate Angular application (`mat-expressive-docs`) that consumes the library as a published package and presents guides, API references, and interactive component demos.

## Language

### Content & Authoring

**Doc Page**:
A single URL in the docs site backed by one markdown file in `public/docs/`. Frontmatter declares the page's title, order, and description.
_Avoid_: Article, guide, document

**Section**:
A top-level grouping of Doc Pages represented as a folder under `public/docs/` (e.g. `getting-started/`, `components/`, `styles-api/`). The folder name is the URL segment.
_Avoid_: Category, group, chapter

**Component Page**:
A Doc Page dedicated to a single library component. Consists of four child-route tabs: Overview, API, Styling, and Playground. Each tab is a separate markdown file (`index.md`, `api.md`, `styling.md`, `playground.md`) within the component's folder.
_Avoid_: Component doc, component guide

**Index Page**:
An optional `index.md` within a Section folder that serves as the Section's landing page. If absent, the router redirects to the first child Doc Page by `order`.
_Avoid_: Landing page, overview page, section home

### Build Pipeline

**Build Script**:
A Node.js/TypeScript script (`scripts/build-docs.ts`) that runs before `ng build`. It uses the TypeScript compiler API to extract library metadata and outputs three artefacts: the API Manifest, the Playground Schema, and the Routes File.
_Avoid_: Pre-build script, generator, compiler

**API Manifest**:
A JSON file (`public/api-manifest.json`) generated by the Build Script. Contains extracted public API metadata for all library exports: selectors, inputs, outputs, JSDoc descriptions, and type signatures.
_Avoid_: API JSON, docs JSON, metadata file

**Playground Schema**:
A JSON file (`public/playground-schemas.json`) generated by the Build Script. Describes each component's inputs with their resolved types, default values, and the control type to render (select, slide-toggle, form-field). Inputs tagged `@internal` in JSDoc are omitted.
_Avoid_: Control schema, input schema, playground config

**Routes File**:
A `routes.txt` file generated by the Build Script, listing one URL per line. Consumed by Angular's `prerender` configuration (`routesFile`) so that every Doc Page is statically pre-rendered during `ng build`.
_Avoid_: Route manifest, URL list, sitemap

### Versioning

**Doc Version**:
A frozen snapshot of the documentation corresponding to a major library release (e.g. v1, v2). Stored under `public/versioned-docs/v1/` in the repository. Served at the `/v1/` URL prefix. The current unreleased state of the docs is never called a version — it is simply "latest."
_Avoid_: Release docs, tagged docs, archived docs

**Latest**:
The documentation reflecting the current `main` branch. Served at the root URL (no version prefix). Always the default landing point for users arriving at the site without a version prefix.
_Avoid_: Current, head, trunk, main docs

**Version Snapshot**:
The act of copying the current `public/docs/` content and generated manifests into `public/versioned-docs/vN/` at major release time. Triggered automatically by a GitHub Actions workflow on the `release: published` event for major-version tags. Committed to `main` before Vercel deploys.
_Avoid_: Archive, freeze, cut, tag

**Version Switcher**:
A compact dropdown in the site header listing all available Doc Versions plus "Latest." Switching navigates to the same page path under the selected version prefix, falling back to the version root if the page does not exist in that snapshot.
_Avoid_: Version picker, version selector, version menu

**Deprecation Banner**:
A sticky warning bar rendered at the top of every Doc Page when the visitor is viewing a non-latest Doc Version. Contains a direct link back to the same page on Latest. Never shown on Latest.
_Avoid_: Version warning, old-version notice, stale banner

### Runtime Features

**Playground**:
An interactive tab on a Component Page that renders the live Angular component alongside auto-generated form controls. Controls are derived from the Playground Schema. Complex cases (projected content, multi-component templates) are handled by a `playground.config.ts` override file in the component's doc folder.
_Avoid_: Demo, interactive preview, sandbox

**Custom Element**:
An Angular component registered via `@angular/elements` (`createCustomElement`) that can be embedded in markdown as a plain HTML tag. The two registered custom elements are `<component-preview>` (renders the Playground) and `<api-reference>` (renders an inline API card from the API Manifest).
_Avoid_: Web component, Angular element (lowercase), widget

**Search Modal**:
A full-screen overlay triggered by `Ctrl+K` / `Cmd+K`. Calls the Pagefind JavaScript API (`pagefind.debouncedSearch()`) and renders results as a list of Doc Page links. Built as a single Angular component using `<mat-form-field>` for the input.
_Avoid_: Search dialog, search popup, command palette

**Page Metadata**:
The set of `<meta>` tags, Open Graph tags, Twitter Card tags, and canonical URL injected into each Doc Page's `<head>`. Managed by `@davidlj95/ngx-meta` via `NgxMetaService.set()`, called after the markdown frontmatter is parsed. Baked into prerendered HTML during SSG.
_Avoid_: SEO tags, head tags, meta description

## Example Dialogue

> **Dev A**: "I'm adding a new Section for animations — should I add it to the Component Page or make a new top-level folder?"
>
> **Dev B**: "New top-level folder under `public/docs/`. A Component Page is specifically for a library component with the four tabs. An animations guide is a Doc Page that lives in its own Section."
>
> **Dev A**: "Got it. Does it need an Index Page?"
>
> **Dev B**: "Only if you want a landing blurb before the individual pages. If the first child Doc Page is enough context, skip the `index.md` and the router will land there automatically."
>
> **Dev A**: "And if I want a live demo on the guide page, I just drop in `<component-preview component="my-component">`?"
>
> **Dev B**: "Exactly — that's a Custom Element. The Build Script already extracted the Playground Schema for that component, so the Playground renders automatically."
