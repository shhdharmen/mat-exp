# mat-expressive Docs Site

The documentation website for the `@ngm-dev/mat-expressive` Angular component library. It is a separate Angular application (`mat-expressive-docs`) that consumes the library as a published package and presents guides, API references, and interactive component demos.

## Language

### Content & Authoring

**Doc Page**:
A single URL in the docs site backed by one markdown file in `public/docs/`. Frontmatter declares the page's title, order, and description.
_Avoid_: Article, guide, document

**Section**:
A top-level grouping of Doc Pages represented as a folder under `public/docs/` (e.g. `getting-started/`, `components/`, `styles-api/`). The folder name is the URL segment.
_Avoid_: Category, group, chapter

**Component Page**:
A Doc Page dedicated to a single library component. Consists of four child-route tabs: Overview, API, Styling, and Playground. The first three tabs are backed by markdown files (`index.md`, `api.md`, `styling.md`); the Playground tab renders a live Angular component with no markdown file.
_Avoid_: Component doc, component guide

**Index Page**:
An optional `index.md` within a Section folder. The router always redirects Section paths to the first child Doc Page by `order`; the Index Page is not used as a landing page.
_Avoid_: Landing page, overview page, section home

### Library Concepts

**Button Family**:
The six `@ngm-dev/mat-expressive` components that share one combinatorial size/shape/appearance/state/selection/variant token system and must render pixel-identical wherever a value overlaps: `MatExpressiveButton`, `MatExpressiveIconButton`, `MatExpressiveFabMenu`, `MatExpressiveFabMenuTrigger` (all `@Directive`s applied to Angular Material's own button/menu components), plus `MatExpressiveButtonGroup` and `MatExpressiveSplitButton` (`@Component`s that deliberately stay on the same global-mixin styling path as the four directives instead of using component-scoped `styleUrls`, so the shared token system isn't forked). `MatExpressiveLoadingIndicator` is not a Button Family member — it has no discrete variant enum and ships ordinary component-scoped `styleUrls`. See `docs/adr/0006-button-family-style-mechanism.md` for the full reasoning.
_Avoid_: Button components, button variants, button suite

### Build Pipeline

**Build Script**:
A Node.js/TypeScript script (`scripts/build-docs.ts`) that runs before `ng build`. It uses the TypeScript compiler API to extract library metadata and outputs three artefacts: the API Manifest, the Playground Schema, and the Routes File.
_Avoid_: Pre-build script, generator, compiler

**API Manifest**:
A JSON file (`public/api-manifest.json`) generated by the Build Script. Contains extracted public API metadata for all library exports: selectors, inputs, outputs, JSDoc descriptions, and type signatures.
_Avoid_: API JSON, docs JSON, metadata file

**Playground Schema**:
A JSON file (`public/playground-schemas.json`) generated by the Build Script. Describes each component's inputs with their resolved types, default values, and the control type to render (select, slide-toggle, form-field). Inputs tagged `@internal` in JSDoc are omitted.
_Avoid_: Control schema, input schema, playground config

**Routes File**:
A `routes.txt` file generated by the Build Script, listing one URL per line. Consumed by Angular's `prerender` configuration (`routesFile`) so that every Doc Page is statically pre-rendered during `ng build`.
_Avoid_: Route manifest, URL list, sitemap

### Versioning

**Doc Version**:
The documentation for a specific major library release (e.g. v1, v2), served at its own subdomain (`v1.expressive.angular-material.dev`). Each Doc Version is a fully independent Vercel branch deployment — it has its own Angular bundle built against the version-accurate library, its own Playground, and its own prerendered HTML. The current unreleased state of the docs is never called a version — it is simply "latest."
_Avoid_: Release docs, tagged docs, archived docs, snapshot

**Latest**:
The documentation reflecting the current `main` branch. Served at the root domain (`expressive.angular-material.dev`). Always the default landing point for users arriving without a subdomain prefix. The Angular app on `main` has no version-awareness — `environment.version` is an empty string.
_Avoid_: Current, head, trunk, main docs

**Version Branch**:
A long-lived Git branch named `N.x.x` (e.g. `1.x.x`, `2.x.x`) following the semantic-release maintenance branch convention. Cut from `main` at the moment a major version is released. Vercel deploys this branch at the corresponding versioned subdomain. Future doc patches for that major version are committed directly to the Version Branch.
_Avoid_: Release branch, versioned branch, snapshot branch

**Version Snapshot**:
The act of cutting a Version Branch from the release tag commit and appending the version identifier to `public/versions.json` on `main`. Triggered automatically by a GitHub Actions workflow on the `release: published` event for major-version tags (`v*.0.0`). No files are copied to `main` — the Version Branch is the snapshot.
_Avoid_: Archive, freeze, copy, commit-to-main

**Version Switcher**:
A compact dropdown in the site header listing all available Doc Versions plus "Latest." Reads the version list from `VersionsService` (which fetches `public/versions.json` from the production domain). Switching performs a full cross-domain redirect to `vN.expressive.angular-material.dev/same-path`, falling back to the version root if the page does not exist.
_Avoid_: Version picker, version selector, version menu

**Deprecation Banner**:
A sticky warning bar rendered at the top of every Doc Page on a versioned subdomain. Visibility is determined by the build-time `environment.version` constant — non-empty means the banner is shown; empty string (Latest) means it is hidden. Contains a direct link back to the same path on Latest. Baked into prerendered HTML at build time, so it is correct on first paint with no client-side URL parsing.
_Avoid_: Version warning, old-version notice, stale banner

### API Reference

**API Reference**:
The section of the docs site dedicated to the library's public API surface. Served under the `/api` path. Distinct from Component Pages — API Reference is data-driven from the API Manifest rather than markdown-driven.
_Avoid_: API docs, API section, reference docs

**API Index Page**:
The landing page for the API Reference at `/api`. Lists all exported symbols from the library, filterable by name, grouped by kind (directive, component, class, interface, type, function, const). Each symbol links to its API Detail Page.
_Avoid_: API list, symbol index, API overview

**API Detail Page**:
A per-symbol page at `/api/{kind}/mat-expressive/{SymbolName}`. Renders the full documentation for one exported symbol: description, JSDoc tag sections (`@deprecated`, `@remarks`, `@example`, `@see`), inputs/outputs tables (for directives and components), structured members (for classes and interfaces), and parameter/return documentation (for functions).
_Avoid_: Symbol page, API card page, reference detail

### Runtime Features

**Playground**:
An interactive tab on a Component Page that renders the live Angular component alongside auto-generated form controls. Controls are derived from the Playground Schema. Complex cases (projected content, multi-component templates) are handled by a `playground.config.ts` override file in the component's doc folder.
_Avoid_: Demo, interactive preview, sandbox

**Search Modal**:
A full-screen overlay triggered by `Ctrl+K` / `Cmd+K`. Calls the Pagefind JavaScript API (`pagefind.debouncedSearch()`) and renders results as a list of Doc Page links. Built as a single Angular component using `<mat-form-field>` for the input.
_Avoid_: Search dialog, search popup, command palette

**Page Metadata**:
The set of `<meta>` tags, Open Graph tags, Twitter Card tags, and canonical URL injected into each Doc Page's `<head>`. Managed by `@davidlj95/ngx-meta` via `NgxMetaService.set()`, called after the markdown frontmatter is parsed. Baked into prerendered HTML during SSG.
_Avoid_: SEO tags, head tags, meta description

## Example Dialogue

> **Dev A**: "I'm adding a new Section for animations — should I add it to the Component Page or make a new top-level folder?"
>
> **Dev B**: "New top-level folder under `public/docs/`. A Component Page is specifically for a library component with the four tabs. An animations guide is a Doc Page that lives in its own Section."
>
> **Dev A**: "Got it. Does it need an Index Page?"
>
> **Dev B**: "Only if you want a landing blurb before the individual pages. If the first child Doc Page is enough context, skip the `index.md` and the router will land there automatically."
>
> **Dev A**: "And if I want a live demo on the guide page?"
>
> **Dev B**: "That's only available on a Component Page's Playground tab — there's no way to embed a live demo inline in a Doc Page's markdown."
